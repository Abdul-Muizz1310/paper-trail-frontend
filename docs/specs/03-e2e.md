# Spec 03 — End-to-end

## Scope

Playwright suite that drives a real browser against the live Next.js dev server, with the backend either (a) mocked via Playwright's `route.fulfill` or (b) pointed at the real deployed backend for the smoke scenario.

Location: `tests/e2e/debate-flow.spec.ts`. Config: `playwright.config.ts` with `baseURL` from `NEXT_PUBLIC_SITE_URL` (defaulting to `http://localhost:3000`).

## Two modes

1. **Mocked mode (default, runs in CI)** — all `**/debates*` requests intercepted and fulfilled from fixtures under `tests/e2e/fixtures/`. Deterministic. Fast.
2. **Live mode (`E2E_LIVE=1`, nightly only)** — hits `https://paper-trail-backend-7h27.onrender.com` directly. Flaky by nature (Render cold starts, LLM variability); not gated on PR merge.

## Core scenario

> User enters a claim → watches the debate arena populate over SSE → sees the verdict → clicks through to the transcript and reads the markdown.

### Fixture setup (mocked mode)

- `POST /debates` → 201 `{ debate_id: "11111111-1111-1111-1111-111111111111", stream_url: "/debates/11111111.../stream" }`
- `GET /debates/{id}/stream` → served by a small handler that pushes a scripted sequence of SSE events over time (state × 3, then done).
- `GET /debates/{id}` → returns three different bodies depending on which call number (first call: 1 round; second: 2 rounds; third: complete with verdict).
- `GET /debates/{id}/transcript.md` → returns a fixture markdown file.

### Test cases

#### Pass

- **P1 — Happy path (mocked)**
  1. Visit `/`.
  2. Fill the claim textbox with "Remote work reduces productivity.".
  3. Click submit.
  4. Assert URL is `/debates/1111...`.
  5. Assert the DebateArena is visible with a "streaming" indicator.
  6. Wait for `<JudgeVerdict>` with `data-verdict="TRUE"` (or whichever the fixture returns) to appear within 10s.
  7. Click the "View transcript" link.
  8. Assert URL ends with `/transcript` and the transcript `<article>` contains a fixture-specific phrase like "Final reasoning".

- **P2 — Deep link to in-progress debate (mocked)**
  1. With the `POST /debates` call skipped, navigate directly to `/debates/1111.../` (mock returns `status: "running"`).
  2. Assert the arena renders and the SSE takes over from the current state.

- **P3 — Transcript deep link (mocked)**
  1. Navigate directly to `/debates/1111.../transcript`.
  2. Assert the markdown body is present and contains expected headings.

#### Failure

- **F1 — Invalid UUID route**
  1. Visit `/debates/not-a-uuid`.
  2. Assert the Next.js 404 page renders; assert no SSE connection is opened (verify via `page.on('request')` filter — zero requests to `**/stream`).

- **F2 — Backend 404 on debate**
  1. Mock `GET /debates/{id}` → 404.
  2. Visit `/debates/1111...`.
  3. Assert the error state renders with a "Go home" link and the link is wired to `/`.

- **F3 — Transcript not found**
  1. Mock `GET /debates/{id}/transcript.md` → 404.
  2. Visit `/debates/1111.../transcript`.
  3. Assert the Next.js 404 page renders.

- **F4 — Submit failure restores input**
  1. Mock `POST /debates` → 500.
  2. Visit `/`, type "my claim", submit.
  3. Assert a toast with role `alert` appears.
  4. Assert the input still contains "my claim" (no data loss).

- **F5 — SSE disconnect + reconnect (mocked)**
  1. Script the SSE handler to close the connection after the first `state` event.
  2. Assert the arena shows a "reconnecting" indicator within 2s.
  3. Reopen the stream and push `done`; assert the verdict appears.

- **F6 — Max retries exceeded**
  1. Script the SSE handler to always 500.
  2. Assert within ~30s the page shows an error state with "connection failed".

## Live smoke (nightly, `E2E_LIVE=1`)

- **L1** Submit the claim "The Eiffel Tower is in Paris." against the real backend.
- Wait up to 90s (Render cold start + debate runtime) for a verdict.
- Assert verdict is `TRUE` and confidence ≥ 0.5.
- **Expected to be flaky** — failure does not block merges; it pages a notification only.
